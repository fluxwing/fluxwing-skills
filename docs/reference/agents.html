Autonomous Agents

Fluxwing’s agents handle multi-step design work, deep validation, and screen composition without manual prompting. Use this guide to decide when to dispatch each agent and what to expect from their output.

Agent Overview

Agents extend Fluxwing beyond single-command interactions. They plan work, execute TodoWrite tasks, and keep your project files aligned with the uxscii standard.

Core Agents

Agent Primary Focus Ideal For
fluxwing-designer End-to-end component + screen creation Design systems, multi-component requests, rapid prototyping
fluxwing-validator Comprehensive quality analysis Pre-release audits, accessibility checks, schema compliance
fluxwing-composer Assemble screens from existing components Storyboarding flows, reusing component libraries

Screenshot Import Agents

The screenshot import workflow is powered by five specialized agents working in concert. These are automatically dispatched by the /fluxwing-import-screenshot command and coordinate to analyze visual designs.

Agent Specialized Role
screenshot-vision-coordinator Orchestrates the analysis pipeline and coordinates other screenshot agents
screenshot-layout-discovery Identifies overall screen structure, sections, and spatial relationships
screenshot-component-detection Locates and classifies individual UI components (buttons, inputs, cards, etc.)
screenshot-visual-properties Extracts visual attributes like dimensions, borders, colors, and text content
screenshot-component-generator Generates uxscii files (.uxm + .md) from analyzed component data
Note: Screenshot agents work automatically during import. You don't dispatch them individually—the coordinator handles orchestration when you run /fluxwing-import-screenshot.

Respecting Data Boundaries

Agents observe the same read/write separation as commands. Understanding that boundary keeps your workspace predictable.

Reference Only

  • {PLUGIN_ROOT}/data/examples/
  • {PLUGIN_ROOT}/data/screens/
  • {PLUGIN_ROOT}/data/docs/
  • {PLUGIN_ROOT}/data/schema/

Project Workspace

  • ./fluxwing/components/
  • ./fluxwing/screens/
  • ./fluxwing/library/

Agents search in the order components → library → bundled examples so custom work always wins over stock templates.

Commands or Agents?

Use commands for focused tasks. Dispatch agents when work requires planning, iteration, or coordinating multiple outputs.

Stick with Commands When You Need:

  • A single component or screen with tight guidance.
  • Quick validation on a specific artifact.
  • A fast scan of available templates.

Dispatch Agents When You Need:

  • Autonomous creation of multi-component systems.
  • Deep quality analysis across dozens of files.
  • Coordinated screen compositions that reuse existing pieces.

fluxwing-designer

The designer agent interprets high-level briefs and turns them into complete component and screen suites. It balances reuse with net-new creation and finishes with a comprehensive status report.

How to Dispatch

Describe the experience you need; Claude Code will suggest the designer agent automatically. You can also request it explicitly: 

Dispatch fluxwing-designer to create a revenue dashboard with metrics, charts, and activity feed.

Execution Phases

  1. Requirements — analyze the request, plan deliverables, build TodoWrite list.
  2. Inventory — check project components, library, and bundled examples.
  3. Systematic Creation — atomic components → composite components → screens.
  4. Quality — validate schema, accessibility, naming, and references.
  5. Reporting — summarize outputs, file paths, validation score, suggestions.

Expect dozens of files created or updated in a single run, each validated and documented.

fluxwing-validator

When your library matures, the validator agent ensures everything still meets Fluxwing’s standards. It reviews ASCII fidelity, schema compliance, accessibility, and metadata completeness, then recommends fixes.

  • Ideal before shipping a new feature or publishing docs.
  • Produces a prioritized checklist with pass/fail details.
  • Pairs well with /fluxwing-validate for spot checks during development.

fluxwing-composer

The composer agent stitches together existing components into coherent screens or flows. It respects your naming conventions, detects gaps, and uses library assets before reaching for bundled templates.

Typical Session

  1. Evaluate requested flow and gather required components.
  2. Confirm assets exist or queue creation via designer agent.
  3. Assemble screen metadata, template, and rendered example.
  4. Return a narrative explaining layout decisions and follow-up actions.

Screenshot Import Workflow

When you run /fluxwing-import-screenshot, the coordinator agent orchestrates a multi-phase pipeline:

  1. Layout Discovery — Identifies screen structure (header, sidebar, main content, etc.)
  2. Component Detection — Locates all UI elements and classifies them by type
  3. Visual Property Extraction — Measures dimensions, extracts text, identifies borders and styles
  4. Component Generation — Creates atomic components first, then composites, then screens
  5. Validation — Ensures all generated files meet uxscii standards

Each specialized agent contributes its expertise, and the coordinator merges results into a unified component library.

Agent Tips

  • Share real data examples so rendered screens feel grounded.
  • Group related asks into one agent dispatch instead of many micro-requests.
  • Run validator after large designer sessions to catch regressions.
  • Leverage ./fluxwing/library/ for reusable variants the composer can pull from.
  • For screenshot imports, use clear, well-lit images focused on single features or screens.
  • Review imported components and customize them to match your brand guidelines.
