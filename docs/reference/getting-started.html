Fluxwing Documentation

Launch Checklist

Install Fluxwing, run your first commands, and learn how uxscii keeps humans and AI in sync. Perfect for new team members or anyone testing the plugin.

1. Install Fluxwing

Fluxwing installs directly inside Claude Code. Add the marketplace, then install the plugin:

/plugin marketplace add fluxwing/claude-code-plugins
/plugin install fluxwing

Once installed, run /fluxwing-create or /fluxwing-library to confirm everything is wired up.

2. Understand the Workflow

Fluxwing uses the uxscii standard—a combination of ASCII art and JSON metadata—to keep designs portable and easy to iterate.

  • ASCII templates capture layout, spacing, and states in human-readable form.
  • .uxm metadata stores IDs, props, accessibility, and relationships.
  • Rendered examples show real data so reviews feel grounded.
Designers, engineers, and AI agents can all read and refine the same artifacts—no proprietary formats required.

3. First Commands to Try

Create a Component

/fluxwing-create button

Fluxwing guides you through props, states, and accessibility before saving files to ./fluxwing/components/.

Browse the Library

/fluxwing-library

Inspect bundled templates and any custom components you’ve already created.

Build a Screen

/fluxwing-scaffold login-screen

Fluxwing composes a full screen, creates any missing components, and outputs rendered examples.

Validate Quality

/fluxwing-validate components/button.uxm

Get instant feedback on schema compliance and accessibility before sharing with the team.

4. What’s Included

  • 11 component templates covering buttons, inputs, cards, modals, navigation, and more.
  • 2 screen examples (login + dashboard) with rendered samples.
  • Comprehensive docs tuned for agent consumption and human review.
  • JSON Schema validation complete with accessibility guidelines.

5. Collaborate Effectively

Fluxwing thrives on rapid conversation loops: AI drafts ASCII, humans give feedback, and the system refines until the design is ready for implementation.

  • Use concrete examples when requesting designs (“billing dashboard with MRR trend”).
  • Review rendered ASCII together during design critiques.
  • Share the ./fluxwing/ workspace via version control to collaborate asynchronously.
